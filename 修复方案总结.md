# py-nl2sql OpenRouter 兼容性修复方案

## 问题总结

经过完整排查，发现以下问题：

### ① Embedding 模型
- **问题**：项目默认使用 `text-embedding-ada-002`（无前缀）
- **OpenRouter 要求**：需要使用 `openai/text-embedding-ada-002` 格式
- **状态**：✅ 已修复

### ② Base URL 配置
- **问题**：系统环境变量可能覆盖 .env 文件
- **OpenRouter 要求**：`OPENAI_BASE_URL=https://openrouter.ai/api/v1`
- **状态**：✅ 已修复（代码层面），⚠️ 需要确保系统环境变量不覆盖

### ③ Chat 模型名称
- **问题**：默认使用 `gpt-4o-mini`（无前缀）
- **OpenRouter 要求**：需要使用 `openai/gpt-4o-mini` 格式
- **状态**：✅ 已修复

### ④ Structured Output (Beta API)
- **问题**：OpenRouter 可能不支持 `client.beta.chat.completions.parse`
- **解决方案**：自动 fallback 到 JSON mode + JSON 解析
- **状态**：✅ 已修复

### ⑤ langchain-openai Embedding 兼容性
- **状态**：✅ 兼容，使用 `OpenAIEmbeddings` 并指定正确的模型名称即可

## 需要额外检查的事项

### ⚠️ 系统环境变量覆盖问题

如果遇到 "No auth credentials found" 错误，可能是系统环境变量覆盖了 .env 文件。

**解决方案**：

1. **检查系统环境变量**（Windows）：
   ```powershell
   $env:OPENAI_API_KEY
   $env:OPENAI_BASE_URL
   ```

2. **临时清除系统环境变量**（仅当前 PowerShell 会话）：
   ```powershell
   Remove-Item Env:\OPENAI_API_KEY -ErrorAction SilentlyContinue
   Remove-Item Env:\OPENAI_BASE_URL -ErrorAction SilentlyContinue
   ```

3. **或者在代码启动前强制设置**（已在代码中实现 `override=True`）

## .env 文件配置

确保 `.env` 文件内容如下：
```
OPENAI_API_KEY=sk-or-v1-你的OpenRouter API Key
OPENAI_BASE_URL=https://openrouter.ai/api/v1
```

## 如果仍然遇到 401 错误

### 检查清单：
1. ✅ `.env` 文件中的 `OPENAI_BASE_URL` 是否为 `https://openrouter.ai/api/v1`
2. ✅ `.env` 文件中的 `OPENAI_API_KEY` 是否为 OpenRouter 格式（`sk-or-v1-...`）
3. ✅ 系统环境变量是否覆盖了 .env（在 PowerShell 中运行上面的检查命令）
4. ✅ OpenRouter 后台中该 API Key 是否有相应模型的权限

### 临时解决方案：
如果系统环境变量仍然覆盖，可以在运行前清除：
```powershell
$env:OPENAI_API_KEY = $null
$env:OPENAI_BASE_URL = $null
python test_nl2sql.py
```
